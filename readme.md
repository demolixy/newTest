java的NIO的学习 

首先我们应该知道为什么使用nio

1 就是我们以前的io操作主要是通过阻塞的方式来实现的，这中方式浪费了很多的资源   所以在jdk1.4之后我们使用nio来操作io，这里的核心就是使用缓冲区

2 常见的几种的nio的数据的格式   就是我们的java中的几种基本格式的数据  但是没有boolean

3 学习nio，首先需要学习几种常量  position limit capacity

常见的buffer的操作  

	在buffer中存在一系列的状态量，这个状态的变量随着写入或者读取的时候肯能会被改变，在缓冲区中可以使用三个值表示缓冲区的状态

	position  表示下一个缓冲区或写入的操作的指针，每一次想缓存区中写入数据的时候指针就会发生改变，指针永远放入到写入的最后一个元素之后。如果写入了四个位置的数据，则position会指向第5个位置

	limit 表示还有多少的数据需要存储或者读取，position《=limit

	capacity 表示缓冲区的最大的容量，limit《=capacity 这个值在分配缓冲区的时候被设置，一般的时候是不可以改变的

	了解通道的作用
		在新的io中通常可以被使用来读取和写入数据，通道类似于之前的输入输出流，但是程序师不会直接操作通道的，所有的内容都是先读入或者写入至缓存区中的，然后再由缓存区中的数据进行操作
		还有一个就是通道于传统的流的操作是不一样的，传统的流操作是分为输入 输出的，但是通道是完成的一个双向的操作，即可以完成输入，也可以完成输出。


	既然通道是一个双向的操作，那么我们的文件就是既可以读取也是可以写入的。
